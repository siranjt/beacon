The umbrella

One watchtower over the entire book.

Before Beacon, every CS workflow was a separate tool — different login, different data shape, different feel. Beacon collapses six of them into one umbrella, with shared auth, shared search, shared memory, and a single AI copilot that travels with you across surfaces.

The umbrella

Six agents under one roof.

Live disengagement signals, growth reports, escalation triage, ICP gating, missed-payment chase, and churn-risk alerts. Each one runs on Beacon's shared spine — same auth, same activity log, same Cmd+K, same copilot.

One auth. Google sign-in. Email-domain allowlist. Three roles: admin, manager, AM.
One search. Cmd+K opens the umbrella palette — every customer, every agent, every page.
One inbox. The launcher aggregates today's queue across all four live agents and surfaces what needs attention.

The brain

Beam asks. Keeper remembers.

Every agent shares one AI copilot — Beam — and one per-customer memory store — Keeper. The same Beam that drafts an email on Customer 360 also queries the book on Customer Beacon, prioritizes the queue on Escalation, and explains a missed-payment row on Miss Payment.

Always on. Press Cmd/Ctrl + K or open the panel from any agent — scope auto-detects.
Cites everything. Every Beam answer carries inline cite chips — click any chip to see the source fact and recency.
Memory carries. A fact written to Keeper on Customer 360 shows up automatically on Escalation, Miss Payment, and the Performance report.

Beam — the copilot

Nineteen tools. Ten scopes. One panel.

Beam is the AI that lives in every Beacon surface. The same panel adapts to where you are — Customer 360, the book, the escalation queue, a Performance report, the missed-invoice list — and surfaces only the tools that make sense on that screen.

19 server-side tools

Read tools (lookup, query book, read notes, read Keeper, get billing, get performance, full-customer-view, mixpanel, reviews, booking history). Write tools (snooze, pin, mark-contacted, add note, add Keeper fact). Draft tools (email, Slack). Every tool execution is audit-logged.

lib/ai/tools/* · audit-by-construction

Per-scope tool subsets

Beam sees only the tools that apply to your current surface — mutators on Customer 360, query-the-book on Manager views, draft tools on every customer-aware scope. The model never reaches for a tool that doesn't fit the context.

SCOPE_TOOL_ALLOWLIST · 10 scopes

Plan-first answers

For multi-step questions, Beam writes a short plan first ("I'll pull the billing record, the Keeper renewal note, and the last 30 days of comms"), then executes the plan tool by tool, citing each source.

prompts.ts · plan-first

Inline citations

Every claim Beam makes ships with a cite chip. [K:integration] means "from Keeper, integration subcategory." [N:note] means "from a customer note." Click any chip to see the exact source row and timestamp.

lib/ai/citations.ts

Grounded refusals

When Beam doesn't have the data, it says so plainly — and logs the gap to beacon_ai_failure_log for the admin gap viewer. No hallucinations. No "best guess" verdicts on questions it can't ground.

/admin/beacon-ai-gaps

Hybrid retrieval

When Beam needs a Keeper fact, it runs Voyage embeddings AND Postgres BM25 in parallel, fuses the candidates with reciprocal-rank fusion, and reranks with Voyage rerank-2.5-lite. Top-5 of ~40 candidates land in context.

lib/brain/retrieve.ts

Keeper — the memory

The truth that travels with the customer.

Keeper holds one canonical row per (customer, topic, field). Five top-level categories. ~25 subcategories. ~70 named structured fields. Every fact carries its source, its timestamp, and its conflict-resolution score.

Five categories · taxonomy v3

Identity (owner, decision-makers, sold-by, AM assignment, business profile). Operational (contract, integration, feature usage, tech stack, renewal, onboarding, performance context). Behavioral (payment pattern, comms preference, seasonal, demo style, competitive context, cadence, sentiment). Concerns (latent risk, next-call agenda, soft red flag). Relationship (advocacy, engagement).

lib/brain/types.ts · ~25 subcategories

Source trust hierarchy

Six provenances: BaseSheet, Chargebee, customer-note, Beam-conversation (AM said it), Beam-extracted (Haiku from notes — must validate), manual. Higher-trust sources auto-confirm at bootstrap; lower-trust sources land in the Validate inbox.

FactSourceType · 6 levels

Conflict resolution

When two facts collide on the same (customer, topic, field), Keeper's deterministic ranker scores both on source trust + recency (60-day half-life) + specificity. The loser stays in the audit log via superseded_by — nothing is ever deleted silently.

lib/brain/ranking.ts

Validate inbox

Haiku extracts candidate facts from AM notes nightly. They land in the Validate inbox tagged with source pills. Managers approve, refine, or reject. Reject-rate is tracked on the admin quality page to keep extraction prompts honest.

/admin/keeper/validate

Derived-from chains

Some facts derive from others ("renewal date" → derived from a contract clause). Keeper tracks the chain via derived_from. When a parent gets superseded, derived children are flagged for re-review — no stale child facts living off retired parents.

SMART-K4 · cascading flags

Stale prune + bootstrap

Facts past their sunset date or below confidence floor auto-prune nightly. New customers get auto-bootstrapped at Stage A — BaseSheet fact write-back populates Keeper on first sight. Every fallback compounds.

The pipeline

What feeds the watchtower.

A four-stage refresh runs against every active customer every day. Stage A every hour. Stages B / C / D every night at 22:00 UTC. Compose every hour to keep customer trends and narrative fresh.

Stage A

Snapshot

0 * * * · hourly

Live Chargebee subs, billing health, entity mapping, Metabase BaseSheet supplement. Integrity assertions guard against silent drops.

Stage B

Comms ingest

0 22 * * · nightly

Bulk comms events from Metabase — App Chat, Email, Phone, Video, SMS. V2 pipeline streams chunks straight into Postgres.

Stage C

HubSpot

0 22 * * · nightly

Company enrichment from HubSpot (contacts, ownership, lifecycle). Soft-fails per object so one outage doesn't break the night.

Stage D

Proactive narrative

0 22 * * · nightly

Deals, calls, AM notes enrichment. Time-budgeted per fetcher so the stage fits inside Vercel's 5-minute function ceiling.

Compose

Trends + narrative

5 * * * · hourly

Updates customer_trends and the per-customer narrative. Skips recently-churned at compose. Auto-bootstrap for new sightings.
